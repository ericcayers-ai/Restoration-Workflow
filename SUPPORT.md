# Support

Thanks for using Restoration Workflow. This is a small open-source project — please
use the channels below so questions stay searchable and security stays private.

## Getting help

| Topic | Where |
|-------|--------|
| Bug in the engine, CLI, API, or UI | [Bug report](https://github.com/ericcayers-ai/Restoration-Workflow/issues/new?template=bug_report.yml) |
| Feature or new model | [Feature request](https://github.com/ericcayers-ai/Restoration-Workflow/issues/new?template=feature_request.yml) |
| Docs unclear or wrong | [Documentation issue](https://github.com/ericcayers-ai/Restoration-Workflow/issues/new?template=documentation.yml) |
| How do I…? (usage) | Prefer a documentation issue or a discussion-style bug report with **How are you running it?** filled in |
| Security vulnerability | [`SECURITY.md`](SECURITY.md) — **never** a public issue |
| Code of Conduct | [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md) — private contact only |

## Before you open an issue

1. Confirm your version (`restore --version`, the Windows launcher banner, or
   `GET /api/health`) — current line is **0.6.0**.
2. Skim [`README.md`](README.md) install notes and [`docs/MODEL_STACK.md`](docs/MODEL_STACK.md)
   if the problem is a missing or gated weight.
3. For Windows portable builds, say whether you used `Run.bat` / the zip from GitHub
   Releases (supported path). A Tauri / native-updater shell is not the supported
   product surface.

## Commercial / SLA support

There is no paid support SLAs or hosted service attached to this repository. Maintainers
respond as time allows.
